# 태그와 브랜치 개요

## 버전과 태그
```
✓ Semantic Versioning 방식
 - 3.34.5 : Major.minor.patch
✓ 태그 생성
  - $ git tag v1.0 1
  - $ git tag -a v1.0.1 -m'msg'
✓ 태그 자세한 정보 확인
 - $ git shwow v1.0.1
✓ 태그 목록 확인
 - $ git tag
✓ 태그 삭제
 - $ git tag -d v1.0.1
```
### 브랜치(Branch)
 - 나무의 가지, 지점
 - 커밋 사이를 가볍게 이동할 수 있는 포인터
```
✓ 깃 브랜치
  - 독립적으로 다시 개발을 진행하는 개념
    ㄴ 동시에 다양한 작업 하게 해줌
    ㄴ 여러 갈래로 나누어서 관리
✓ 장점
 - 영향 없이 새로운 기능 개발/ 버그 수정
 - 안전하게 실험 가능
```
```
✓ 기본 브랜치
 - 처음 만들어지는 브랜치 : main
 - 기본 설정 : $ git config --global init.defaultBranch main
 - 이름 수정 : $ git branch -M newBname
 ```
 ```
 ✓ 브랜치 확인
  - $ git branch : 커밋이 발생한 브랜치 목록 보이기
  - $ git branch -v : 브랜치마다 마지막 커밋 ID와 메세지도 함께 표시
  - '* main' = 현재 작업하고 있는 브랜치 표시
  - 이동 = $ git switch [bname] / $ git checkout [bname]
  - $  git checkout -b develop : 브랜치 develop 생성 후 HEAD 이동
  - $ git branch [-d | --delete] [branchName] / $ git branch -D [branchName] : 지정된 것을 삭제(병합된) / (병합되지 않아도 ) 삭제
  - 병합 여부 = --merged와 --no-merged
  - $ git branch -h : 도움말 보기
  ```
