# 비교 개요

## 깃 3 영역
```
✓ PC의 탐색기 저장소 폴더에서 보이는 파일 : 작업 디렉토리의 파일
✓ 저장소 하부의 .git 폴더에서 정보로 숨겨진 파일 : 스테이징 / 깃 (지역) 저장소
✓ 리눅스 명령어 ls로 보이는 파일 : 작업 디렉토리의 파일
✓ 깃 명령 git ls-file로 보이는 파일 : 스테이징 영역
```
### 1-1 파일 비교
 - 오른쪽 영역이 기준
 ```
 ✓ git diff
 ✓ $ git diff --staged[HEAD]
 ✓ $ git diff HEAD
 ✓ $ git diff HEAD~2
 ✓ $ git diff HEAD~2 HEAD~
 ✓ $ git diff HEAD~ HEAD
 ✓ $ git diff HEAD~2 HEAD
 ```
### 1-2 파일 삭제
```
✓ $ rm [file] : 작업 디렉토리에서 file 삭제
✓ $ git rm [file] : 작업와 스테이징 영역에서 모두 삭제
✓ $ git rm --cached [file] : 작업 디렉토리에서는 삭제 안됨, 스테이징 영역의 파일 목록 표시
✓ $ git restore f : 스테이징 영역의 상태를 작업 디렉토리에 복원
✓ $ git restore --staged f : 깃 저장소 상태를 스테이징 영역에 복원
✓ $ git restore --source=HEAD --worktree f or HEAD f : 깃 저장소 상태를 작업 디렉토리에 복원
✓ $ git restore --source=HEAD --staged --worktree f : 깃 저장소 상태를 스테이징 영역과 작업 디렉토리에 함께 복원
```
